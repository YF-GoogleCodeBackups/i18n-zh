Debian安装程序是Debian官方的操作系统安装组件。它提供了Debian操作系统的一系列安装方式。您适合用何种安装方式将取决于您的计算机硬件架构。

您可以借助Debian website上的安装指南找齐lenny的安装光盘映像。

安装手册也可以在Debian官方CD/DVD的第一张盘上找到，路径是： 

/doc/install/manual/language/index.html

您也可以登陆errata以查看关于Debian安装程序的已知问题列表。

3.1. 安装环境有哪些新特性？

自从名为etch的第一个官方发布以来，Debian安装程序就在不断地发展，在硬件支持上有显著改进，还增加了很多有趣的新功能。

在发布说明中我们只列举安装程序的主要变更。如果您想对从etch开始各项变更的详细情况有个大致的了解，请阅读lenny的发布声明。beta和RC版可以在关于Debian安装程序的网页news history上看到。

3.1.1. 主要更新

更灵活的分区方法

现在可以创建文件系统，在一个 LVM 卷上使用向导进行分区。

安装程序也可以创建加密的文件系统。通过使用手动分区，您可以在dm-crypt和loop-aes之间选择，通过passphrase加密密钥或随机密钥，您可以设置很多其它选项。使用向导分区方式，则安装程序会创建一个加密的 LVM 磁盘分区，它以逻辑分区形式包含了其它文件系统（/boot除外）。

图形用户界面

如果您更喜欢用图形用户界面，可以尝试加参数 installgui来启动安装程序。

图形安装程序的功能几乎和标准安装程序相同，只不过界面不同而已。这里有一项例外：图形前端不支持使用随机按键创建加密分区的功能。

图形用户界面的主要优势在于它比标准用户界面(newt)支持更多语言。图形安装程序的相关信息以及它同标准安装程序最重要的不同点被以文档形式记录在安装手册的附录中。

注意

图形用户界面可能对部分计算机架构不适用。

修复模式

您可以用这个安装程序来解决现有系统的问题，例如系统无法启动。第一步就像标准安装一样，只不过安装程序不使用分区软件，而提供一个关于系统修复的选单。

启动安装程序并使用rescue选项，或在启动参数中添加rescue/enable=true以进入修复模式。

用sudo命令替代以root账号登陆

在专家安装模式下您可以选择不创建root账号（它将被锁定），而设置sudo从而使第一个用户可以使用sudo命令进行系统管理。

简单邮件配置

标准系统安装完成后，安装程序将会对系统内的邮件服务程序作基本设置，这将使其只能完成本地邮件系统的架设，邮件服务程序将会对同一网络下的其它系统不可用。如果您想配置系统以便管理更多系统（发送邮件或接收邮件），您就需要在安装结束后重新配置邮件系统。

选择桌面环境

如果用户请求安装，程序将默认安装一个GNOME桌面环境。

然而，希望安装多个桌面环境的用户可以通过增加参数（tasks=standard, kde-desktop安装KDE，tasks=standard, xfce-desktop安装Xfce）而很容易地做到。注意：当从CD映像安装并且不添加网络镜像作为额外软件源时，此方法无效；从DVD映像安装或其它方式时上述方法可用。

也有单独的CD映像是以KDE或Xfce作为默认桌面环境的。

新增语言支持

得益于翻译人员的巨大努力，Debian现在可以在文本界面下以47种语言安装。这比 etch版多了六种语言。此次增加的语言包括白俄罗斯语、世界语、爱沙尼亚语、库尔德语、马其顿语、塔加路族语、越南语、沃尔夫语。由于缺少对应的翻译更新，有两个语言在本版中不再被支持：波斯语和威尔士语。

如果使用图形用户界面，将会有十一种额外语言可选。由于它们的字符集无法在非图形界面显示，这些语言只有在图形安装程序中才可以被选到。这些新增语言是：孟加拉语，不丹语，古吉拉特语，印地语，格鲁吉亚语，高棉语，马来语，尼泊尔语，旁遮普语，泰米尔语和泰国语。

不想使用任何地区设置的用户可以在安装程序的语言选择项中选C作为他们所喜欢的地区设置。关于系统所支持的语言种类的信息可以在d-i languages list看到。

简化的地区和时区选择

语言、国家和时区的设置步骤被简化，以减少用户需要提供的数据量。安装程序将根据所选语言推测系统的所在国家和所处时区，无法确定时会提供一个短小的选单。如果需要的话，用户仍然可以采用模糊组合。

改进的系统全面本地化

大多数以前由localization-config工具处理的国际化和本地化任务，现在被包含在Debian安装程序或软件包中。这意味着选择一种语言就会在标准和桌面环境中自动安装这一语言所需的软件包（字典，文档，字体... ）。纸张尺寸配置和一些高级X Window键盘设置中对有些语言的配置项将不再被自动处理。

注意：仅当在安装环境下可用时，跟对应语言相关的软件包才会被自动安装。

3.1.2. 自动安装

前一节中所提到的大量变更也意味着安装程序的使用预配置文件进行自动安装支持的调整。这意味着，如果您有能在etch安装程序中正常使用的预配置文件，您不能指望它能在新安装程序中正常使用，而无需修改。

有一则好消息是Installation Guide现在有一个关于预设置使用的带文档扩充的单独附录。

lenny发布的安装程序介绍了一些令人兴奋的新特性，它们允许更深入、更容易的自动安装。它也新增了对 RAID, LVM 和加密文件系统的高级磁盘分区支持。 LVM详情见相关文档。

3.1.3. Install Debian GNU/Linux with a braille display

You can install Debian GNU/Linux 5.0 (lenny) with a braille display. The braille display must be connected to a USB or a serial port. If your braille display is connected to a USB port and the American braille table is supposed to be used, you can press Enter in the boot menu. The screen reader BrlTTY is standardized to search for braille displays at the USB port and uses the American Braille table. If the braille display is connected to a serial port or if you want to use a different braille table, you have to press the Tab key in the boot menu first. Then, you can configure the screen reader BrlTTY with the kernel parameter brltty.

For the parameter, the following syntax is valid: 

brltty=driver,device,table

All operands are optional.

Driver

The driver for the employed braille display. Here, you must enter either a code consisting of two letters or the word auto. If this argument is not entered, the automated recognition is activated by default.

Device

The device can be entered as relative to /dev/ as well as an absolute specification.

Table

Defines the braille table for the desired language. By default, the US table is employed.

3.1.3.1. Examples

brltty=ht,ttyS0,de

The Handy Tech Driver is used. The braille display is connected to Com1. The German braille table is used. 

brltty=,,de

Here, only the German braille table is specified. Therefore, BrlTTY will try to find a braille display at a USB port.

3.2. 流行度统计

安装环境会再次提供popularity-contest软件包的安装选项。在etch版中这个软件包默认状态是不安装的，而在早期版本中是要安装的。

popularity-contest向Debian项目提供了关于发布中哪些软件包被实际使用的宝贵信息。这些信息主要用于决定哪些软件包应当被包含在安装光盘上，而Debian开发人员也常查看它们以决定要不要采用不再有维护者的软件包。

popularity-contest的信息收集过程是匿名进行的。如果您能参与这项调查，帮我们改进Debian，我们会非常高兴。
