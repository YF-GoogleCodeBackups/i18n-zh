Debian 安装程序是 Debian 官方的安装系统。它提供了 Debian 操作系统的各种安装方式。您适合用何种安装方式，取决于您的计算机硬件架构。

您可以借助 Debian 网站上的安装指南找到 lenny 的安装程序映像。

安装手册也可以在 Debian 官方 CD/DVD 的第一张盘上找到，路径是： 

/doc/install/manual/language/index.html

您也可以检查勘误表，以查看 Debian 安装程序的已知问题列表。

3.1. 安装系统有哪些新特性？

自从在 etch 中的第一个官方发布以来，Debian 安装程序有许多开发工作，在硬件支持上有显著改进，还增加了很多有趣的新功能。

在发布说明中，我们只列举安装程序的主要变更。如果您想对从 etch 开始各项变更的详细情况有个大致的了解，请阅读 lenny 之 beta 和 RC 版本的发布声明，它们位于 Debian 安装程序的新闻网页上。

3.1.1. 主要更新

更灵活的分区方法

现在可以使用分区向导在 LVM 卷上创建文件系统。

安装程序也可以创建加密的文件系统。通过使用手动分区，您可以使用密码或随机按键在 dm-crypt 和 loop-aes 之间选择，还可以调谐其它各种选项。使用向导分区方式，则安装程序会创建加密的 LVM 磁盘分区，它以逻辑卷的形式包含了其它文件系统(/boot 除外)。

图形用户界面

如果您更喜欢用图形用户界面，可以尝试用 installgui 启动安装程序。

图形安装程序的功能几乎和标准安装程序相同，只不过界面不同而已。这里有一项例外：图形前端不支持使用随机按键创建加密分区的功能。

图形用户界面的主要优势在于，它比标准用户界面(newt)支持更多语言。图形安装程序的相关信息，以及它同标准安装程序最重要的不同点，被以文档形式记录在安装手册的附录中。

注意

图形用户界面可能对部分计算机架构不适用。

拯救模式

您可以用这个安装程序来解决现有系统的问题，例如系统无法启动。第一步就像标准安装一样，只不过安装程序不使用分区软件，而提供一个关于系统修复的选单。

启动安装程序并使用 rescue 选项，或在启动参数中添加 rescue/enable=true 以进入修复模式。

用 sudo 命令替代以 root 账号登陆

在专家安装模式下您可以选择不创建 root 账号(它将被锁定)，而设置 sudo，从而使第一个用户可以使用 sudo 命令进行系统管理。

简单邮件配置

“标准系统”安装完成后，安装程序将会对系统内的邮件服务程序作基本设置，这将使其只能完成本地邮件系统的架设，邮件服务程序将会对同一网络下的其它系统不可用。如果您想配置系统以便管理更多系统(发送邮件或接收邮件)，您就需要在安装结束后重新配置邮件系统。

选择桌面环境

如果用户请求安装，程序将默认安装 GNOME 桌面环境。

然而，希望安装多个桌面环境的用户可以通过增加参数(tasks="standard, kde-desktop" 安装 KDE，"tasks=standard, xfce-desktop" 安装 Xfce)而很容易地做到。注意：当从 CD 映像安装，并且不添加网络镜像作为额外软件源时，此方法无效；从 DVD 映像安装或其它方式时上述方法可用。

也有单独的 CD 映像是以 KDE 或 Xfce 作为默认桌面环境的。

新增语言支持

得益于翻译人员的巨大努力，Debian 现在可以在文本界面下以 47 种语言安装。这比 etch 版多了六种语言。此次增加的语言包括白俄罗斯语、世界语、爱沙尼亚语、库尔德语、马其顿语、塔加路族语、越南语、沃尔夫语。由于缺少对应的翻译更新，有两个语言在本版中不再被支持：波斯语和威尔士语。

如果使用图形用户界面，将会有十一种额外语言可选。由于它们的字符集无法在非图形界面显示，这些语言只有在图形安装程序中才可以被选到。这些新增语言是：孟加拉语，不丹语，古吉拉特语，印地语，格鲁吉亚语，高棉语，马来语，尼泊尔语，旁遮普语，泰米尔语和泰国语。

不想使用任何地区设置的用户可以在安装程序的语言选择项中选 C 作为他们所喜欢的地区设置。关于系统所支持的语言种类的信息可以在 Debian 安装程序语言列表 看到。

简化的地区和时区选择

语言、国家和时区的设置步骤被简化，以减少用户需要提供的数据量。安装程序将根据所选语言推测系统的所在国家和所处时区，无法确定时会提供一个短小的选单。如果需要的话，用户仍然可以采用模糊组合。

改进的系统全面本地化

大多数以前由 localization-config 工具处理的国际化和本地化任务，现在被包含在 Debian 安装程序或软件包中。这意味着选择一种语言就会在标准和桌面环境中自动安装这一语言所需的软件包(字典，文档，字体 ... )。纸张尺寸配置和一些高级 X Window 键盘设置中对有些语言的配置项将不再被自动处理。

注意：仅当在安装环境下可用时，跟对应语言相关的软件包才会被自动安装。

3.1.2. 自动安装

前一节中所提到的大量变更，也意味着安装程序的使用预配置文件进行自动安装支持的调整。这意味着，如果您有能在 etch 安装程序中正常使用的预配置文件，您不能指望它能在新安装程序中正常使用，而无需修改。

有一则好消息是安装指南现在有一个关于预设置使用的带文档扩充的单独附录。

lenny 安装程序引进了一些令人兴奋的新特性，它们允许更深入、更容易的自动安装。它也新增了对 RAID，LVM 和加密 LVM 卷的高级磁盘分区支持。

3.1.3. 使用盲文显示器安装 Debian GNU/Linux

您可以使用盲文显示器安装 Debian GNU/Linux 5.0 (lenny)。盲文显示器需要连接在 USB 或者串口上。如果您的盲文显示器连接在 USB 口上并且要使用美式盲文表，您可以在启动菜单中按 Enter。屏幕阅读器 BrlTTY 会遵从标准在 USB 端口寻找盲文显示器并使用美式键盘表。如果盲文显示器连接到串口或者您想使用不同的盲文表，您需要在启动菜单中先按 Tab 键。然后，您可以使用内核参数 brltty 配置屏幕阅读器 BrlTTY。

至于参数，以下语法是合法的： 

brltty=driver,device,table

所有选项都是可选的。

driver

盲文显示器的驱动。这里，您必须输入两个字母组成的代码或者单词 auto。如果没有给出参数，会默认开启自动识别。

device

设备可以输入相对于 /dev/ 的相对路径或者绝对路径。

table

定义目标语言的盲文表。默认情况下，使用美式表。

3.1.3.1. 例子

brltty=ht,ttyS0,de

使用便捷科技驱动(Handy Tech Driver)。盲文显示器连接到 Com1。使用德文盲文表。 

brltty=,,de

这里，只指定了德文盲文表。因此，BrlTTY 将会尝试在 USB 端口寻找盲文显示器。

3.2. 流行度竞赛

安装环境会再次提供 popularity-contest 软件包的安装选项。在 etch 版中这个软件包默认状态是不安装的，而在早期版本中是要安装的。

popularity-contest 向 Debian 项目提供了关于发布中哪些软件包被实际使用的宝贵信息。这些信息主要用于决定哪些软件包应当被包含在安装光盘上，而 Debian 开发人员也常查看它们以决定要不要采用不再有维护者的软件包。

popularity-contest 的信息收集过程是匿名进行的。如果您能参与这项调查，帮我们改进 Debian，我们会非常高兴。
